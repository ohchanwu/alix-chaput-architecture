## 프로젝트 소개

제가 처음으로 제대로 만든 웹 프로젝트로, 2025년 중반에 작업했습니다. 의도적인 연습 과제로서 HTML, CSS, JavaScript를 모두 직접 손으로 작성했습니다 — 프레임워크도, 빌드 단계도, 의존성(dependency)도 없습니다.

## 기능

- **라이트 / 다크 모드** 전환
- **7개의 프로젝트 페이지**, 각 페이지마다 자체 이미지 갤러리 포함
- **반응형 내비게이션** — 데스크톱에서는 메인 내비게이션 바, 작은 화면에서는 햄버거 메뉴
- Bahnschrift, DIN Pro, Fenwick를 사용한 **커스텀 타이포그래피**

## 스택

순수 HTML, CSS, 그리고 바닐라(vanilla) JavaScript. GitHub Pages에서 호스팅됩니다.

## 배운 점

- **WebP가 사진에서 무조건 이득은 아니다.** JPEG를 WebP로 변환하면 사진 콘텐츠의 경우 오히려 _더 큰_ 파일이 만들어질 수 있습니다 — 일괄 변환으로 저장소(repo) 용량이 불어난 뒤에야 이 사실을 알게 되었습니다.
- **Safari는 Chrome과 미묘하게 다르게 렌더링한다.** Safari에서만 나타난 레이아웃 결함을 계기로, 크로스 브라우저 테스트를 마지막에 한 번 점검하는 것이 아니라 평상시 작업 흐름의 일부로 삼게 되었습니다.
- **실제 성능 개선은 이미지 용량에서 나온다.** 디자인 도구가 내보내는(export) 결과물은 보통 실제로 배포해야 할 것보다 훨씬 무겁습니다. 이미지 크기 조정과 압축은 필수입니다.
- **비용이 이득을 넘어서면 범위를 줄여라.** 이 사이트는 원래 이중 언어(bilingual)로 기획되었습니다 — `/en/`에는 아직 부분적인 영어 버전이 남아 있습니다. Alix와 저는 뒤늦게 이 사이트의 대상 사용자에게는 전체를 번역할 만한 가치가 없다고 판단했고, 그래서 라이브 사이트는 프랑스어로만 제공됩니다.
